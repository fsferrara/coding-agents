# Introduction

This handbook exists because adopting coding agents in a team is not purely a tooling question — it is an engineering practice question. Choosing a tool is easy. Knowing how to integrate it into your workflow, how to write tasks that agents can act on, how to review agent-produced code, and how to reason about what agents can and cannot do reliably: these require foundations that most guides skip.

## Who this book is for

Engineers and tech leads who are evaluating or already using coding agents in a professional software development context. No prior experience with AI is required, but a working familiarity with software development workflows is assumed.

## How to read it

The three parts build on each other but can be read independently:

- **[Part 1: Process Foundations](part1-process/README.md)** grounds coding agents in the process context they operate in.
- **[Part 2: AI Fundamentals](part2-ai-fundamentals/README.md)** gives you a working mental model of the underlying AI technology.
- **[Part 3: Coding Agents](part3-coding-agents/README.md)** covers coding agents specifically: what they are, how they work, and how to configure and use them effectively.

The [Appendices](appendices/README.md) include a glossary of terms and a guide mapping chapters to companion files.

## Companion files

This book is hosted in a public repository. The [`examples/`](../examples/) directory contains skills, configurations, and prompt templates referenced from individual chapters. See the [Repository Guide](appendices/repository-guide.md) for a mapping of chapters to companion files.
